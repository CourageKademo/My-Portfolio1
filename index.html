Front-End

It refers to the process of building the part of a website or web application that users directly see and interact with — the layout, visual design, buttons, forms, animations, and everything that makes the interface usable across devices

Back-end

Is the work done "behind the scenes" of a web or mobile application — it involves server-side logic, databases, and APIs that users don’t see directly. A back-end developer builds the systems that store, retrieve and process data, handle business rules

Full-stack

Is the practice of building both the front-end (what the user interacts with) and the back-end (the logic, database and server) of an application, meaning a developer can take a project from the user interface right through to data storage and business logic.

UMLX

Graphical modelling and transformation languageused in model-driven software engineering. It provides a higher-level visual syntax for specifying how models (for example UML diagrams) should be transformed into other models or into executable artifacts.
